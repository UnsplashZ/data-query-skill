# v0.1.1 安装与回归闭环修复方案

## 背景

`v0.1.0` release zip 可以安装，`manifest` 也能通过校验，但在解压后的安装包内运行完整 eval 会失败：

```text
FileNotFoundError: evals/fixtures/data-query-work
```

根因是 `validate_manifest.py` 把任意路径段里的 `data-query-work` 都当作生成目录忽略，导致 `evals/fixtures/data-query-work/knowledge/` 没有进入 manifest，也没有进入 release zip。源码 checkout 上的 eval 能通过，但安装包内 eval 不可执行。

## 目标

`v0.1.1` 必须把安装、配置、回归和发布验收连成闭环：

- release zip 解压后可以直接运行完整 offline eval，不抛 traceback。
- manifest 覆盖 eval 依赖，不再漏掉 `evals/fixtures/data-query-work/knowledge/`。
- release/package 验收必须解压 zip 后再跑一遍 eval。
- 已有本机配置可增量补源，不覆盖已有 ClickHouse/Metabase/MySQL 等凭证。
- 安装后有一键 post-install 状态检查，区分 `installed`、`configured`、`connected`、`query_verified`。
- README/SKILL 明确升级已有安装、补缺失源、重启 Codex。

## P0 修复

### 1. 修复 manifest/package fixture 缺失

改 `scripts/validate_manifest.py`：

- `data-query-work/` 只作为仓库顶层生成目录忽略。
- 不忽略 `evals/fixtures/data-query-work/knowledge/`。
- 继续忽略任意层级 `__pycache__`、`.pyc`。
- 重生成 `manifest.json`，把 `evals/fixtures/data-query-work/knowledge/` 文件纳入 release 包。

验收：

```bash
python3 scripts/validate_manifest.py
python3 scripts/package_skill.py --json
```

解压 zip 后必须存在：

```text
evals/fixtures/data-query-work/knowledge/manifest.yaml
```

### 2. 让 eval 缺 fixture 时返回清晰 FAIL

改 `scripts/eval_skill_pack.py`：

- `eval_knowledge_capture_write_validate_search` 在 copy fixture 前检查路径。
- 缺 fixture 时返回 `EvalResult(..., FAIL, "missing fixture ...")`，不得抛 traceback。

### 3. 解压包后复测

改 `scripts/eval_skill_pack.py`：

- `package_zip_contract` 不只检查 zip 结构，还要把 zip 解压到临时目录。
- 在解压目录内运行：

```bash
python scripts/validate_manifest.py
python scripts/scan_sensitive_info.py .
python scripts/eval_skill_pack.py --allow-blocked --skip-package-contract
```

新增 `--skip-package-contract`，避免 eval 自己递归打包自己。

`--skip-package-contract` 只能跳过“打包并递归 eval 包”这一项，不能跳过 manifest、敏感扫描、知识库、setup、Metabase mock 等普通 eval。

## P0 配置体验

### 4. setup_connections 增量合并

改 `scripts/setup_connections.py`：

- 新增 `--merge`。
- 新增 `--add-sources odps,mysql`，作为 `--sources odps,mysql --merge` 的便捷入口。
- `--add-sources` 与 `--sources` 同时出现时直接失败，避免列表合并语义不清。
- 已有 output 时：
  - 默认仍拒绝写入，避免误覆盖。
  - `--overwrite` 仍全量替换。
  - `--merge` 只添加缺失的 `engine/profile`，不覆盖已有 profile。
- 非交互模式下，缺失源写 placeholder。
- 交互模式下，只对缺失源提示输入；已有源跳过。

验收：

- 已有 ClickHouse/Metabase 配置，执行 `--add-sources odps,mysql --non-interactive` 后，已有 section 内容保持不变。
- 用哨兵值证明已有 profile 未被覆盖。
- 缺失 ODPS/MySQL 被添加。
- 不使用 `--merge` 或 `--overwrite` 时，已有 output 仍失败。

## P1 安装自动化

### 5. 新增 post_install_check

新增 `scripts/post_install_check.py`，统一输出：

- package files：`SKILL.md`、`manifest.json`、`scripts/setup_connections.py`
- manifest status
- sensitive scan status
- config path
- configured sources
- missing sources
- offline smoke
- real connection smoke 是否跳过
- 状态分级：
  - `installed`
  - `configured`
  - `connected`
  - `query_verified`
- 最后固定提示：

```text
Restart Codex to pick up new skills.
```

默认不做真实连接；加 `--real-smoke` 才调用真实连接 smoke。

CLI 参数：

- `--config <path>`：指定本机配置。
- `--json`：输出机器可读 JSON。
- `--offline-ok`：安装和离线检查通过即可返回 0，即使缺配置或未做真实连接。
- `--real-smoke`：尝试真实连接 smoke，失败时按失败状态输出。

退出码：

- 缺包文件、manifest 失败、敏感扫描失败、离线 smoke 失败：非 0。
- `--offline-ok` 下，缺配置或未做真实连接不导致非 0。

## P1 文档

更新 README/SKILL：

- 安装后可运行 `python scripts/post_install_check.py`。
- 升级已有安装：
  1. 保留本机 `~/.internal-data-query/data-sources.yaml`。
  2. 覆盖 skill 包。
  3. 运行 post-install。
  4. 用 `setup_connections.py --add-sources ...` 补缺失源。
  5. 重启 Codex。
- 明确配置可解析不等于真实可查询。

## 发布验收

发布前必须通过：

```bash
python3 scripts/validate_manifest.py
python3 scripts/scan_sensitive_info.py .
python3 scripts/eval_skill_pack.py --allow-blocked
python3 scripts/package_skill.py --json
```

并解压 release zip 后再次通过：

```bash
python3 scripts/validate_manifest.py
python3 scripts/scan_sensitive_info.py .
python3 scripts/eval_skill_pack.py --allow-blocked
python3 scripts/post_install_check.py --offline-ok
```

GitHub Actions：

- `release.yml` 和 `release-please.yml` 必须安装 `requirements.txt` 后再跑 eval。
- `release.yml` 的 release notes 应提示 `post_install_check.py --offline-ok`、增量补源和重启 Codex。
