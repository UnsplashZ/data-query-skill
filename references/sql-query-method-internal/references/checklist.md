# SQL 查询任务检查清单

## 接需求时

- [ ] 指标/实体明确
- [ ] 时间范围明确
- [ ] 粒度明确：订单、用户、日期、SKU、营期、班级等
- [ ] 过滤条件明确
- [ ] 输出形式明确：SQL、结果表、Excel、Feishu、报告
- [ ] 未明确但会影响结果的问题已追问或注明假设

## 写 SQL 前

- [ ] 已判断 ODPS / ClickHouse / 其他数据源
- [ ] 已查已有 skill / runbook / 旧 SQL / schema KB
- [ ] 已确认候选表
- [ ] 已确认关键字段存在
- [ ] 已确认时间字段、金额字段、状态字段含义
- [ ] 已确认 join key 和数据粒度

## 执行前

- [ ] 方言语法符合当前引擎版本
- [ ] ODPS 有必要的分区过滤
- [ ] ClickHouse 没有使用不兼容的新语法
- [ ] 比率计算做了除零保护
- [ ] 日期边界为明确的 inclusive/exclusive
- [ ] JOIN 不会意外放大指标

## 验证

- [ ] 小范围/LIMIT 查询已跑通
- [ ] 枚举值或状态分布已检查
- [ ] 行数、金额、用户数有 sanity check
- [ ] 正式范围已执行或明确说明未执行
- [ ] 结果和 SQL 没有被包装成“已验证”的假象

## 交付

- [ ] 给出可直接执行 SQL 或结果文件路径
- [ ] 说明数据源、主表、关键字段
- [ ] 说明业务口径和假设
- [ ] 说明验证方式
- [ ] 说明剩余不确定性
