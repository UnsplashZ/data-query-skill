# 声乐前端渠道退费率对比：3月 BD1-KOL 异常判断

## 触发场景
用户质疑：不同渠道规模差异很大，不能用绝对退费额判断渠道质量，应该按渠道 GMV 分母计算退费率；同时追交接率是 4月才开始，3月 M0/M1 高需要找其他原因。

## 核心修正
- 渠道归因必须同时看：`退费率`、`GMV规模`、`退费额贡献`。
- 绝对退费额只能说明影响规模/处理优先级，不能单独说明渠道质量。
- 对比不同渠道时，使用同支付月、同 lag 口径：`渠道 lag 退费额 / 渠道支付月 GMV`。
- 对小分母渠道要设规模阈值或单独标记，避免小样本高退费率误导。
- 当 3月 M0 已经高时，不应把 4月追交接率作为主解释；应先看支付月成交质量、渠道/投手、价格带、原因结构。

## 本次验证口径
范围：声乐 + 前端 + 课程 + 全款/尾款。
时间：支付月 2025-10 至 2026-04。
M0/M1：`dateDiff('month', pay_month, refund_month)`。
分母：支付月内该渠道 GMV。
分子：对应 lag 的退款金额，来自退款表 refund_time。

## 关键发现
BD1-KOL 月度退费率：

| 支付月 | M0退费率 | M1退费率 |
|---|---:|---:|
| 2025-10 | 1.41% | 1.82% |
| 2025-11 | 0.39% | 3.34% |
| 2025-12 | 1.31% | 2.44% |
| 2026-01 | 1.33% | 1.83% |
| 2026-02 | 0.47% | 2.56% |
| 2026-03 | 4.91% | 2.49% |
| 2026-04 | 2.77% | 未成熟 |

解释：
- 3月 BD1-KOL M0 明显异常，不是单纯规模大造成的。
- 3月 BD1-KOL M1 退费额大，但退费率不是历史最高，也不是所有渠道最高；M1 需要和工作室1-信息流、析出-企微&公众号一起看。
- 3月 M0 渠道层面，BD1-KOL 属于“高退费率 + 大 GMV + 大退费额”的优先排查对象。

3月 M0 有规模渠道退费率示例：
- 工作室1-中小：5.94%，GMV 10.64万，退费额 0.63万
- BD1-KOL：4.91%，GMV 244.81万，退费额 12.03万
- 直播-自然流：4.88%，GMV 45.15万，退费额 2.20万
- BD2-图书：4.68%，GMV 25.24万，退费额 1.18万

BD1-KOL 3月价格带：
- M0：2980档 5.03%，1880档 4.85%，1880以下 4.19%
- M1：1880档 3.64%，1880以下 2.32%，2980档 2.08%

BD1-KOL 3月投手（GMV>=5万）：
- M0：汪国炳 5.79%，李聿为 5.51%，杜思博 2.44%
- M1：杜思博 2.84%，李聿为 2.59%，汪国炳 1.16%

## 报告写法建议
避免写：
- “BD1-KOL 退费最多，所以渠道最差。”
- “M1 高说明4月交接动作导致退费。”

推荐写：
- “按渠道 GMV 归一后，3月 BD1-KOL M0 退费率 4.91%，显著高于其过去数月 M0 水平；同时该渠道 GMV 244.81万，因此既是质量异常点，也是退费额主要贡献来源。”
- “3月 M1 不是 BD1-KOL 独占异常。BD1-KOL 退费额最大，但退费率 2.49%，低于工作室1-信息流、工作室1-中小、析出-企微&公众号等渠道，需要分开判断 M0 即时问题和 M1 滞后释放。”
- “追交接率是4月动作，不能解释3月 M0 高。3月 M0 应优先回到成交来源、投手、价格带、承诺边界和学员约束筛查。”
