# 声乐后端退费明细：3月以来与3/4月支付月份追踪

## 适用场景

用户要求重导声乐后端退费明细，并追踪某些退费月份的退款“是什么时候来的 / 当时支付规模怎么样”。

本次验证样例：
- 范围：`refund_time >= '2026-03-01 00:00:00'` 至当前
- 来源：ClickHouse `tock_dwd_order_refund_df`
- 维度：声乐后端退费，关联 `dim_camp_df` 补营期阶段/轨次
- 输出文件：`~/.hermes/output/query_results/vocal_backend_detail/声乐后端退费明细_3月以来_含3月4月来源追踪_20260511_111855.xlsx`
- 生成脚本：`~/.hermes/python/projects/vocal-backend-detail/export_vocal_backend_refund_since_mar_with_trace.py`

## 核心口径

明细筛选：
```sql
FROM tock_dwd_order_refund_df r
LEFT JOIN dim_camp_df d ON r.camp_id = d.camp_id
WHERE r.refund_time >= toDateTime('{START_DATE} 00:00:00')
  AND r.refund_time < today() + 1
  AND r.cci3_name = '声乐'
  AND r.new_front_end_name LIKE '%后端%'
  AND r.refund_amount > 0
```

业务分类沿用旧口径：
- `main_first_level = '课程'` -> `课程续费`
- `main_first_level = '电商'` -> `电商`
- `main_first_level = '权益'` 且二级类目/商品名命中文旅活动关键词 -> `文旅/活动权益`
- 其余权益 -> `权益其他`

## 关键新增追踪表

当用户问“3月4月退费都是什么时候来的”，不要只看 `refund_month`。要按 **退费月份 × 支付月份** 追踪：

```python
mar_apr = detail[detail['退费月份'].isin(['2026-03', '2026-04'])]
refund_by_pay_month = (
    mar_apr.groupby(['退费月份', '支付月份'], dropna=False)
    .agg(
        退费单数=('订单流水号', 'nunique'),
        退费人数=('用户ID', 'nunique'),
        退费金额=('退费金额', 'sum'),
        对应支付金额=('支付金额', 'sum'),
        平均支付到退费天数=('支付到退费天数', 'mean'),
    )
    .reset_index()
)
refund_by_pay_month['退费金额/对应支付金额'] = refund_by_pay_month['退费金额'] / refund_by_pay_month['对应支付金额']
```

建议输出 sheet：
- `3月4月_退费来自支付月份`
- `3月4月_支付月份×阶段分类`
- `3月4月_支付月份×原因Top200`
- `3月4月_支付月份×商品Top200`
- `3月4月_金额段×支付月份`

## 输出与验证要求

- Excel 必须包含中文表头，金额/人数为数值单元格，不要写成“万”或百分比文本。
- 如果用户说“更新到截止昨天的数据”，用右开区间固定到今天 00:00:00，例如：
  - `END_DATE_EXCLUSIVE = datetime.now().strftime('%Y-%m-%d')`
  - SQL 条件：`refund_time < toDateTime('{END_DATE_EXCLUSIVE} 00:00:00')`
  - 口径说明写成：`{START_DATE} 至 {END_DATE_EXCLUSIVE} 前，按退费时间过滤（截止昨天）`
  - 不要继续使用 `today() + 1`，否则会包含今天数据。
- 写出后用 `openpyxl.load_workbook(..., read_only=True, data_only=True)` 重开校验：
  - sheet 名与行数
  - `退费明细` 行数
  - `退费金额` 样例单元格类型为数值类型
  - `退费明细` 的最大 `退费时间` 不晚于目标截止日（例如截止昨天时，最大值应小于今天 00:00:00）
- 汇报时说明是否重新查库、文件路径、总量校验、核心业务判断。

## 常见误读

- `对应支付金额` 是这批已退订单自身的支付金额，不是完整支付 cohort 分母；不能直接当作真实退费率分母。
- “3/4月退费从什么时候来”应看 `pay_month`，不是只看 `refund_month`。
- 3月与4月机制不同：3月偏历史单释放，4月偏3月支付单次月释放。不要合并成一个泛泛的“近期退费高”。
