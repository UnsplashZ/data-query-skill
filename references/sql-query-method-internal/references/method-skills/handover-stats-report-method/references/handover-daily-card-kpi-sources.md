# 交接数据看板卡片 KPI 口径修正（2026-05）

用于飞书交接数据看板/互动卡片这类 5 SKU 汇总卡片，不替代完整交接统计报告口径。

## 触发场景

当用户要求生成或修复「交接数据看板」卡片，尤其涉及：
- 开课率
- 加微率
- 选期率
- 旧字段 `selected_wechat`
- 飞书互动卡片按钮分页回调

## 当前正确口径

### 开课率
- 来源：飞书表《营期明细》
- 筛选：`开课节点 = 开课封板`
- 分子：`已开课人数`（用户明确说明为正价课开课人数）
- 分母：`订单数`
- 注意：不要用 `衔接课开课人数 / 订单数` 作为卡片主展示的开课率。

### 加微率
- 来源：飞书表《营期明细》
- 筛选：`当前节点 = 加微封板`
- 分子：`已加微人数`
- 分母：`订单数`

### 选期率
- 来源：飞书表《营期明细》
- 筛选：`当前节点 = 加微封板`
- 分子：`已选期人数`
- 分母：`订单数`

## 旧字段兼容

`selected_wechat` / “选期加微率” 不再是卡片主展示指标。若旧卡片按钮或旧回调路径仍请求该字段，只作为 legacy fallback 兼容：

- 优先从 debug 中的 `total_wechat / total_selected` 推导；
- debug 不完整时再 fallback 到 `current_wechat.value`；
- 不应写回主缓存作为正式指标。

主展示保持三项：
- `bridge_open`：开课率（但含义已改为正价课开课人数 / 订单数）
- `current_wechat`：加微率
- `select`：选期率

如果修复卡片：
1. 先确认飞书表头实际列名和节点字段，不要假设旧范围足够。`开课节点` 可能在 `AG` 之后，读取范围要覆盖到该列。
2. 加微/选期聚合继续用 `当前节点 = 加微封板`。
3. 开课聚合要单独用 `开课节点 = 开课封板`，不要复用加微封板聚合结果。
4. 团队选期排行等原有基于加微封板的排名逻辑不要被开课封板口径误改。
5. 修改后至少验证：
   - `python -m py_compile /Users/zheng/.hermes/scripts/handover_team_report_card.py`
   - 读取飞书表并打印各 SKU 的分子、分母、百分比；
   - 若涉及按钮回调，重启相关 gateway 后点击测试卡按钮，确认日志出现 `Patched handover daily card ... to sku=...` 且不再出现 `selected_wechat` 构建错误。

## 常见错误

- 把“开课率”继续算成 `衔接课开课人数 / 订单数`。
- 三个主指标全部从加微封板同一批数据算，忽略开课率应使用开课封板。
- 为兼容旧按钮而把 `selected_wechat` 重新写进主缓存，导致卡片又显示“选期加微率”。
- 只发卡片不点击测试按钮，误以为互动回调已验证。
